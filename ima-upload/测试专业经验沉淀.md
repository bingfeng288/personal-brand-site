## 二、CI/CD 基础建设经验

### Jenkins 多环境部署

- **环境数量**：4 套（开发 / 测试 / 预发 / 生产）

### 流水线设计要点

2. **环境隔离**：各环境独立配置，生产环境需人工审批
3. **测试集成**：自动化测试（Robot Framework）嵌入流水线
4. **通知机制**：构建失败 → 企业微信 / 钉钉通知

| 语言     | Python                                        |

### UI 自动化实践经验

- 页面对象模型（POM）设计模式
- 元素定位策略（ID / CSS / XPath）
- 数据驱动测试（Data-Driven Testing）
- 失败截图 + 日志追踪

### 遇到的坑与解决方案

| 问题                 | 解决方案                                     |
| -------------------- | -------------------------------------------- |
| 元素加载慢导致失败   | 显式等待（Explicit Wait）                    |
| 动态 ID 无法定位     | CSS 选择器 + 父元素遍历                     |
| 跨浏览器兼容性       | 多浏览器 Driver 配置                        |
| 小程序 WXML 约束     | 避免 JS 方法调用，使用 WXS 处理            |

## 五、测试报告与验收经验

### 政务项目验收特点

- 必须附正式测试报告
- 报告需符合工信部 / 人社部相关规范
- **软件评测师证书**是专业资质背书

### 测试报告核心要素

1. **测试范围**：明确测什么、不测什么
2. **测试环境**：硬件、软件、网络配置
3. **测试用例**：覆盖率统计（功能点覆盖 + 代码覆盖）
4. **缺陷统计**：按严重程度分布（致命 / 严重 / 一般 / 轻微）
5. **测试结论**：通过 / 不通过，遗留风险说明

### AI 加速测试报告生成

- 用 Claude / WorkBuddy 辅助生成测试用例
- 用 AI 自动提取缺陷统计并生成图表
- 目标：把 2 天报告的工时压缩到 0.5 天

## 七、AI 工具在测试中的应用经验

### 已实践的 AI 工具

| 工具          | 用途                                     | 效果                        
| ------------- | ---------------------------------------- | --------------------------- |
| Claude        | 测试用例生成、Bug 分析报告辅助           | ⭐⭐⭐⭐                      |
| WorkBuddy    | 代码辅助、自动化脚本生成                 | ⭐⭐⭐⭐                      |
| Cursor        | 自动化测试代码编写                       | ⭐⭐⭐⭐                      |
| AgentScope   | 多智能体协作（规划 / 执行 / 校验）      | ⭐⭐⭐（探索中）             |
| Hermes        | 创意实现辅助                             | ⭐⭐⭐                        |

### AI 测试工作流设计

```
需求文档
   ↓ (AI 解析)
测试用例草稿
   ↓ (人工审核 + 补充边界条件)
最终测试用例
   ↓ (AI 生成自动化脚本框架)
Robot Framework 脚本
   ↓ (执行 + AI 分析失败原因)
测试报告草稿
   ↓ (AI 润色 + 图表生成)
最终测试报告
```

### AI 工具的局限性（实测）

- 计算结果不稳定（如 toFixed 类数值处理）
- AI 可靠性边界需人工把控
- 生成的测试用例需人工补充边界条件
- **结论**：AI 是辅助，不能替代人工判断
