- 已提交推送：`2355aa7 feat: 聊天页面支持 MiMo API 格式`

## AI 咨询聊天页面 — 401 修复 + 密钥安全

- **401 根因**：`chat.html` 中 `settings.authPrefix || 'Bearer '` 对空字符串 `""` 错误兜底为 `"Bearer "`，导致 MiMo 收到 `api-key: Bearer tp-cyx...`（密钥前多了 `Bearer `）
- **修复**：改用 `??`（仅过滤 `undefined`/`null`，保留空字符串）
- **密钥安全**：用户要求请求体中 Key 不可明文
  - 方案1（理想）：Key 只存在边缘函数中，前端只发 `{ messages }` → 新版 `functions/chat.js` 已写好，但 MCP deploy_folder 工具持续报 "meta file is missing or empty" 和 "API error: undefined"，无法部署
  - 方案2（当前）：Key 在前端 Base64 分段混淆存储，源码中无明文，运行时 `atob()` 解码后发送 → 兼容旧边缘函数格式
- **MCP 部署问题**：
  - `functions/` + `meta.json` → "Edge functions exist but meta file is missing or empty"
  - `edge-functions/` + 无 meta → 同样报错
  - `static` 模式 → "API error: undefined"
  - CLI 部署需要登录（`edgeone login`）
  - **待办**：用户需手动去 EdgeOne Pages 控制台更新边缘函数代码，实现真正的服务端密钥存储
- 边缘函数新版本代码已就绪（`_getConfig()` 配置内嵌，前端只发 `messages`），等 MCP 部署修复后即可切换
